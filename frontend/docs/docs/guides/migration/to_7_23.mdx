### Изменения в сборке {#changes-build}

- Добавлена поддержка Java 17
- Версия Spring Boot изменена на 2.5.12

### Изменения в XML API {#changes-xml-api}

- Упразднены устаревшие схемы: `page-2.0`, `region-1.0`

- Появилась новая схема `application-3.0`. В данной схеме добавлены новые атрибуты `path` и `subtitle` для элемента `<sidebar>`.
Также добавлен новый элемент `<sidebars>`.
Чтобы перейти на новую схему нужно перенести элемент `<sidebar>` во внутрь элемента `<sidebars>`. Таким образом добавлена поддержка нескольких
`<sidebar>`, которые будут отображаться в зависимости от url страницы, который должен соответствовать атрибуту `path`.

*Пример как работало в схемах application-1.0 и application-2.0*

```xml
<?xml version='1.0' encoding='UTF-8'?>
<application xmlns="http://n2oapp.net/framework/config/schema/application-1.0">
    <sidebar src="test" class="test"
            title="Test Sidebar" home-page-url="http://google.com/">
        <nav ref-id="testMenu"/>
    </sidebar>
</application>
```

*Пример как работает в схеме application-3.0*
```xml
<?xml version='1.0' encoding='UTF-8'?>
<application xmlns="http://n2oapp.net/framework/config/schema/application-3.0">
    <sidebars>
        <sidebar path="/home"
                 src="home" class="home"
                 title="Home Sidebar" home-page-url="http://google.com/">
            <nav ref-id="homeMenu"/>
        </sidebar>
        <sidebar path="/profile"
                 src="profile" class="profile"
                 title="Profile Sidebar">
             <nav ref-id="profileMenu"/>
        </sidebar>
    </sidebars>
</application>
```

- Добавлен новый атрибут `datasource` для элемента `<sidebar>`. С помощью него можно ссылаться на источник данных `<datasource>`,
заданный в файле *.application.xml. Указанный источник данных в `<sidebar>` позволяет разрешать по контексту страницы
значения атрибутов `title` и `subtitle`, а также `value` для параметров запроса `<a>` и `<open-page>` в `<menu-item>`.
Для использования этого функционала необходимо указать в `path` элемента `<sidebar>` некоторый параметр и также установить
его в качестве `value` для фильтра `<datasource>`.

*Пример использования источников данных в `<sidebar>`*
```xml
<?xml version='1.0' encoding='UTF-8'?>
<application>
  <datasources>
    <datasource id="person" query-id="person">
      <filters>
          <eq field-id="id" value=":person_id"/>
      </filters>
    </datasource>
  </datasources>

  <sidebars>
    <sidebar path="/person/:person_id/**"
             datasource="person"
             title="{fio}">
      <nav>
        <menu-item name="Документы">
          <open-page page-id="docs" route="/person/:person_id/docs">
            <path-param name="person_id" value=":person_id"/>
            <query-param name="fio" value="{fio}" datasource="person"/>
          </open-page>
        </menu-item>
      </nav>
    </sidebar>
  </sidebars>
</application>
```
